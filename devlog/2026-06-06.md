### 安全审查与改进（基于全栈代码审查报告）

#### 第 1 轮：P0 底线修复 ✅

| # | 改进项 | 文件 | 说明 |
|---|--------|------|------|
| A | 数据库竞态条件修复 | `backend/db.js` | 增加 Promise guard (`dbReady`)，多个并发请求共享同一个初始化 Promise，防止创建多个数据库实例导致数据丢失 |
| B | 数据库崩溃保护 | `backend/db.js` | `scheduleSave()` / `saveNow()` 中 `try/catch` 包裹写盘操作，防止磁盘满/权限错误崩溃进程 |

| E | JWT 硬编码密钥 | `backend/middleware/auth.js` | 移除 fallback secret `'dev-secret-change-me-in-production'`，未配置 `JWT_SECRET` 时 `process.exit(1)` 拒绝启动 |
| F | 邀请码安全 | `backend/utils/invite.js` | `Math.random()` → `crypto.randomInt()`，邀请码密码学安全 |
| G | Error Boundary | `frontend/src/components/ErrorBoundary.jsx` | 新增 React Error Boundary 组件，包裹在 `<App>` 外层，捕获子组件渲染错误并显示恢复 UI |
| H | Auth 严格限流 | `backend/index.js` | Auth 路由增加独立限流器（20 次/15min），防止暴力破解登录 |
